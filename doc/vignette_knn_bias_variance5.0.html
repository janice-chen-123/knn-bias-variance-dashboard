Monte Carlo Study of the MSE Decomposition in k-NN Regression

Jiachen Chen, Yiwen Zhao, Yuxi Ren, Jintong Li

Overview

This vignette demonstrates a minimal Monte Carlo framework to study the mean squared error (MSE) decomposition for k-nearest neighbors (k-NN) regression. The main purpose is to show how bias, variance, and prediction error behave under a known data-generating process.

We work under a known data generating process (DGP):

\[ Y = f(X) + \varepsilon, \qquad X \sim \mathrm{Unif}(0,1)^d,\quad \varepsilon \sim \mathcal{N}(0,\sigma^2). \]

The goal is to estimate (pointwise on a grid) the components:

  • Bias: \(\mathrm{Bias}(x) = E[\hat f(x)] - f(x)\)
  • Variance: \(\mathrm{Var}(\hat f(x))\)
  • MSE for estimating the signal: \(\mathrm{MSE}_f(x) = \mathrm{Bias}(x)^2 + \mathrm{Var}(\hat f(x))\)
  • MSE for predicting outcomes: \(\mathrm{MSE}_Y(x) = \mathrm{MSE}_f(x) + \sigma^2\)

Main functions used in this vignette

This vignette uses several helper functions from the package. Each function has a different role in the simulation workflow.

  • dgp_x() generates the predictor matrix \(X\) from a uniform distribution on \([0,1]^d\).
  • dgp_f() evaluates the true regression function \(f(x)\) under either the 1D or 2D setting.
  • dgp_y() generates the response variable \(Y\) by adding random noise to the true signal.
  • knn_predict() computes k-NN regression predictions for a set of test points.
  • make_grid() creates the evaluation grid on which the decomposition is studied.
  • mc_knn_decomp() is the main simulation function. It repeats data generation and prediction many times and returns the pointwise bias, variance, and MSE quantities.
  • mc_knn_curve() computes the average MSE across a set of candidate values of \(k\).
  • plot_mse_1d() visualizes the pointwise decomposition results in the one-dimensional case.
  • mc_knn_curve_components() summarizes the grid-averaged bias, variance, and MSE components for each candidate \(k\).
  • plot_curve_components() plots how these averaged components change across different values of \(k\).

Together, these functions provide a simple simulation framework for studying the bias-variance tradeoff in k-NN regression.

DGP and target functions

We support two target functions:

  1. Baseline (1D): \(f(x) = \sin(2\pi x)\)
  2. Dimensionality extension (2D): \(f(x_1, x_2) = \sin(\sqrt{x_1^2 + x_2^2})\)

Monte Carlo bias–variance decomposition (1D)

Below is a small Monte Carlo run (smaller B for vignette speed; the project default is typically B=500). Next, we run mc_knn_decomp(), which is the main function for the Monte Carlo study. For each repetition, it generates a training sample, applies k-NN regression, predicts on a fixed grid, and then summarizes the pointwise bias, variance, and MSE. In this vignette, we use a smaller value of \(B\) to keep the running time reasonable.

res_1d <- mc_knn_decomp(
  n = 200, k = 15, B = 200,
  sigma = 0.2, d = 1, type = "1d",
  grid_size = 101, seed = 123
)

Assumption Check

In our simulation, the predictor variable \(X\) is generated independently, and the noise term \(\epsilon\) is also generated independently with mean zero and constant variance. This ensures that the observations satisfy the independence and constant variance assumptions required for bias–variance decomposition.

The true regression function is smooth, which makes k-NN regression appropriate for estimating the conditional mean function. Therefore, the assumptions underlying our simulation and analysis are satisfied.

Visualizing the decomposition

We next visualize the pointwise bias–variance decomposition across the input space. For each grid location \(x\), the Monte Carlo experiment estimates Bias\(^2(x)\), Var\((\hat f(x))\), and the total prediction error \(\text{MSE}_Y(x)\). As expected, \(\text{MSE}_Y(x) \approx \text{Bias}^2(x) + \text{Var}(\hat f(x)) + \sigma^2\), and boundary regions may show larger errors due to fewer neighbors on one side.

plot_mse_1d(res_1d, show = c("bias2", "variance", "mse_y"))

Comparing f(x) (mean prediction) to the true f(x)

This plot overlays the true regression function \(f(x)\) with the Monte Carlo estimate of the mean k-NN predictor \(\mathbb{E}[\hat f(x)]\) at each grid point. The gap between the two curves visualizes systematic error (bias): larger deviations indicate higher bias for the chosen \(k\).

library(ggplot2)

pw <- res_1d$pointwise
df <- data.frame(x = pw$x1, f_true = pw$f_true, f_hat_mean = pw$f_hat_mean)

ggplot(df, aes(x = x)) +
  geom_line(aes(y = f_true, color = "True f(x)"), linewidth = 1.2) +
  geom_line(aes(y = f_hat_mean, color = "Monte Carlo mean E[f_hat(x)]"), linewidth = 1.2) +
  scale_color_manual(values = c(
    "True f(x)" = "#0000FF",
    "Monte Carlo mean E[f_hat(x)]" = "#FF00FF"
  )) +
  labs(
    title = "True regression function vs Monte Carlo mean of k-NN predictions",
    x = "x", y = "Value", color = NULL
  ) +
  theme_minimal(base_size = 14) +
  theme(legend.position = "top")

Bias–Variance tradeoff across k

To examine the full bias-variance tradeoff, we use mc_knn_curve_components(), which extends mc_knn_curve() by keeping the separate averaged components instead of only the total MSE.

For each candidate \(k\), the function:

  • runs mc_knn_decomp() for \(B\) Monte Carlo repetitions,
  • computes pointwise Bias\(^2\), Variance, and MSE on the evaluation grid,
  • averages these quantities over the grid.

This output makes it easier to see how increasing \(k\) usually decreases variance but increases bias. We then use plot_curve_components() to visualize these averaged components across candidate values of \(k\).

curve_comp <- mc_knn_curve_components(
  k_values = k_values,
  n = 200, B = 200, sigma = 0.2,
  d = 1, type = "1d",
  grid_size = 101, seed = 10
)

plot_curve_components(curve_comp, show = c("avg_bias2", "avg_variance", "avg_mse_y"))

2D illustration

The 2D case uses the same functions; predictions are evaluated on a grid on \([0,1]^2\). For a vignette, use a smaller grid to keep runtime reasonable.
