Monte Carlo Study of the MSE Decomposition in k-NN Regression

Jiachen Chen, Yiwen Zhao, Yuxi Ren, Jintong Li

Overview

This vignette demonstrates a minimal Monte Carlo framework to study the mean squared error (MSE) decomposition for k-nearest neighbors (k-NN) regression.

We work under a known data generating process (DGP):

\[ Y = f(X) + \varepsilon, \qquad X \sim \mathrm{Unif}(0,1)^d,\quad \varepsilon \sim \mathcal{N}(0,\sigma^2). \]

The goal is to estimate (pointwise on a grid) the components:

  • Bias: \(\mathrm{Bias}(x) = E[\hat f(x)] - f(x)\)
  • Variance: \(\mathrm{Var}(\hat f(x))\)
  • MSE for estimating the signal: \(\mathrm{MSE}_f(x) = \mathrm{Bias}(x)^2 + \mathrm{Var}(\hat f(x))\)
  • MSE for predicting outcomes: \(\mathrm{MSE}_Y(x) = \mathrm{MSE}_f(x) + \sigma^2\)

DGP and target functions

We support two target functions:

  1. Baseline (1D): \(f(x) = \sin(2\pi x)\)
  2. Dimensionality extension (2D): \(f(x_1, x_2) = \sin(\sqrt{x_1^2 + x_2^2})\)

Assumption Check

In our simulation, the predictor variable \(X\) is generated independently, and the noise term \(\epsilon\) is also generated independently with mean zero and constant variance. This ensures that the observations satisfy the independence and constant variance assumptions required for bias–variance decomposition.

The true regression function is smooth, which makes k-NN regression appropriate for estimating the conditional mean function. Therefore, the assumptions underlying our simulation and analysis are satisfied.

Visualizing the decomposition

We next visualize the pointwise bias–variance decomposition across the input space. For each grid location \(x\), the Monte Carlo experiment estimates Bias\(^2(x)\), Var\((\hat f(x))\), and the total prediction error \(\text{MSE}_Y(x)\). As expected, \(\text{MSE}_Y(x) \approx \text{Bias}^2(x) + \text{Var}(\hat f(x)) + \sigma^2\), and boundary regions may show larger errors due to fewer neighbors on one side.

plot_mse_1d(res_1d, show = c("bias2", "variance", "mse_y"))

Comparing f(x) (mean prediction) to the true f(x)

This plot overlays the true regression function \(f(x)\) with the Monte Carlo estimate of the mean k-NN predictor \(\mathbb{E}[\hat f(x)]\) at each grid point. The gap between the two curves visualizes systematic error (bias): larger deviations indicate higher bias for the chosen \(k\).

pw <- res_1d$pointwise
plot(pw$x1, pw$f_true, type = "l", xlab = "x", ylab = "Value",
     main = "True f(x) vs Monte Carlo mean of k-NN predictions")
lines(pw$x1, pw$f_hat_mean)
legend("topright", legend = c("f(x)", "E[ f_hat(x) ]"), lty = 1, bty = "n")

Bias–Variance tradeoff across k (new helper)

To visualize the classic bias–variance tradeoff, we scan a set of candidate values of k and compute the grid-averaged decomposition components.
The helper function mc_knn_curve_components() is a wrapper around mc_knn_decomp():

  • For each k in k_values, it runs a Monte Carlo experiment (B replicates),
  • Computes pointwise Bias², Variance, and MSE on a prediction grid,
  • Then averages these quantities over the grid to produce one row per k.

This provides a compact summary suitable for plotting the tradeoff curves versus k.
We then use plot_curve_components() to plot selected components (e.g., avg_bias2, avg_variance, avg_mse_y).

# A more informative scan: average Bias^2, Variance and MSE_Y vs k
curve_comp <- mc_knn_curve_components(
  k_values = k_values,
  n = 200, B = 200, sigma = 0.2,
  d = 1, type = "1d",
  grid_size = 101, seed = 10
)

plot_curve_components(curve_comp, show = c("avg_bias2", "avg_variance", "avg_mse_y"))
